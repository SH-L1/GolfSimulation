# 골프 코칭 애플리케이션: 3D 스윙 시각화 아키텍처

## 1. 개요

본 문서는 2D/3D 환경에서 추출된 **17개의 희소(Sparse) 관절 좌표 데이터**를 기반으로, 모바일 환경에서 전문가 수준의 골프 스윙을 3D 아바타로 실시간 시각화하기 위한 시스템 설계 및 기술적 해결 방안을 정의합니다.

무거운 애니메이션 파일(FBX)을 직접 구동하는 대신, 표준화된 3D 아바타 에셋에 경량화된 좌표 데이터를 실시간으로 덮어씌우는 **절차적 애니메이션(Procedural Animation)** 기법을 채택하여 모바일 서비스에 최적화된 환경을 구축합니다.

---

## 2. 핵심 과제 및 기술적 해결 방안

### A. 누락된 신체 부위 및 미세 관절 복원

> **문제**: 17개의 기본 좌표만으로는 골프 스윙의 핵심인 척추의 꼬임과 손가락의 세밀한 그립을 표현할 수 없습니다.

#### 해결 방안

| 기법 | 설명 |
|------|------|
| **역운동학(IK) 적용** | 추출된 손목과 발목의 좌표를 목표 지점(Target)으로 설정하여, 엔진이 해부학적 한계 내에서 팔꿈치와 어깨의 위치를 자연스럽게 역산 |
| **X-Factor (상하체 분리) 연산** | 양 어깨 벡터와 양 골반 벡터의 방향성 차이(X-Factor)를 도출하고, 이를 바탕으로 데이터에 존재하지 않는 척추와 가슴의 회전값을 보간하여 적용 |
| **정적 포즈 결합** | 골프 그립을 쥐고 있는 정적 마스킹 애니메이션을 손목 하단에 영구 적용한 뒤 클럽을 동적으로 부착하여 이질감 해소 |

### B. 프레임 간 끊김(Stuttering) 현상 극복

> **문제**: 추출된 데이터의 프레임레이트(예: 30fps 이하)와 모바일 기기의 렌더링 환경(60fps 이상) 사이의 격차로 인해 동작이 끊겨 보일 수 있습니다.

#### 해결 방안

**구면 선형 보간(Slerp)** 기법을 활용하여 이전 프레임과 다음 프레임 사이의 빈 공간을 수학적 곡선으로 부드럽게 메워줍니다. 이를 통해 상대적으로 낮은 주사율의 데이터가 유입되더라도 사용자 화면에서는 매끄러운 **60fps 이상의 동작**으로 재생됩니다.

```
데이터 프레임(30fps)    ●───────────────●───────────────●
                         ↓   Slerp 보간   ↓
렌더링 프레임(60fps)    ●───●───●───●───●───●───●───●───●
```

### C. 관절 떨림(Jittering) 노이즈 제거

> **문제**: 비전 기반 추적 기술의 한계로 인해 발생하는 프레임 단위의 미세한 좌표 떨림 현상이 3D 모델에 그대로 전이됩니다.

#### 해결 방안

시각화 엔진으로 데이터를 전달하기 직전, **동적 로우패스 필터(One Euro Filter)**를 적용합니다.

- 빠른 스윙 궤적: 지연 최소화
- 정지/느린 동작 구간: 미세 떨림 강력 보정

## 3. 시스템 데이터 파이프라인

동작 생성 및 렌더링 과정은 다음과 같은 **단방향 스트림**으로 설계됩니다.

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  데이터 정규화   │───▶│ 최적 궤적 도출   │───▶│ 실시간 렌더링    │
│                 │    │                 │    │                 │
│ 골반 중심 원점화 │    │ 스윙 시퀀스 생성  │    │ 뼈대 매핑 + 보간  │
│ 신체 비율 배제   │    │ 경량 데이터 출력  │    │ 아바타 동작 완성  │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 3.1 데이터 정규화

개별 프로의 신체 비율 차이를 배제하기 위해 **골반 중심을 원점**으로 데이터를 정규화합니다.

### 3.2 최적 궤적 도출

정규화된 좌표와 계산된 물리적 지표(척추각 등)를 활용해 도출된 최적의 스윙 시퀀스를 **경량 데이터 포맷**으로 출력합니다.

### 3.3 실시간 렌더링

모바일 기기에 탑재된 표준 아바타의 뼈대 계층 구조에 해당 시퀀스 데이터를 **실시간으로 매핑 및 보간**하여 동작을 완성합니다.

## 4. 렌더링 엔진 선정: Unity

해당 서비스를 구현함에 있어 Unreal Engine 대비 **Unity**가 가지는 우위는 다음과 같습니다.

| 항목 | 설명 |
|------|------|
| **앱 이식성 (UaaL)** | Unity as a Library 기능을 통해 React Native, Flutter, 혹은 네이티브 OS(iOS/Android)로 개발된 앱 내부에 3D 뷰어 화면을 충돌 없이 가볍게 삽입 가능 |
| **경량화 및 배터리 효율** | 모바일 환경에 최적화된 빌드 사이즈를 제공하여 앱 초기 다운로드 용량 절감, 구동 시 발열 및 배터리 소모 최소화 |
| **백엔드 통신 편의성** | 서버로부터 실시간 스트리밍되는 데이터를 파싱하고 아바타의 관절 제어 시스템에 연결하는 과정이 직관적이고 효율적 |
