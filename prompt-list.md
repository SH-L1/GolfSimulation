## 반복 패턴 (자동 적용 규칙)

### 패턴 1: Phase 완료 후 딥 분석 + 보고서
- **트리거:** "다음 Phase 진행해도 좋아" 또는 Phase 구현 완료 시
- **자동 동작:**
  1. Phase 구현
  2. 폴더 전체를 깊이 읽고 동작 원리 파악
  3. `research.md`에 상세 보고서 작성 (구조, 흐름도, 통계, 세팅 가이드, 제한사항)
  4. `plan.md` 체크리스트 업데이트 (✅ 마킹)

### 패턴 2: 버그 수정 시 리뷰 + 디버그 로그
- **트리거:** 오류/버그 보고 + 수정 요청
- **자동 동작:**
  1. 깊이 분석 → 근본 원인 파악
  2. 수정 적용
  3. `Review.md`에 분석 과정 및 수정 내용 기록
  4. 콘솔 `Debug.Log` 메시지 추가 (정상/오류 시나리오 모두)

### 패턴 3: 유니티 설정 안내
- **트리거:** 새 컴포넌트/시스템 추가 후
- **자동 동작:** Unity Editor에서 필요한 세팅을 체크리스트 형태로 안내

### 패턴 4: 이름/컨벤션 일관성
- **트리거:** 에셋/메뉴 이름 불일치 발견 시

### 패턴 5: plan.md 계획 추가
- **트리거:** 사용자가 미래 계획을 언급할 때
- **자동 동작:** `plan.md`에 해당 계획 섹션 추가 (미구현 항목은 [ ]로)

## 사용자 선호 사항

- Phase 완료 후 항상 `research.md`에 상세 보고서 작성
- 버그 수정 시 항상 `Review.md`에 분석 과정 기록 + `Debug.Log` 추가
